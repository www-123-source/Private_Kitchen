# 执行策略文档

## 概述

为了确保 Private Kitchen 应用更新的顺利进行，同时避免上下文累积问题，我们采用混合策略来执行实现计划。该策略将14个任务分成多个阶段，每个阶段完成后清理上下文并提交代码，然后进入下一阶段。

## 混合策略说明

### 策核心理念
1. **分组执行**：将相关任务组合在一起，保持上下文的连贯性
2. **定期清理**：每个阶段完成后清理会话历史，避免累积
3. **持续集成**：每个阶段都提交代码，确保进度可追溯
4. **模块化开发**：按功能模块划分，便于管理和测试

### 执行原则
- 保持每个阶段的上下文清晰（3-4个任务一组）
- 每个阶段完成后必须提交代码
- 阶段之间可以休息和重新规划
- 优先处理核心功能，逐步完善细节

## 执行流程

### 开始新阶段
1. 打开实现计划文档
2. 确认当前阶段的任务范围
3. 使用子代理驱动方式
4. 逐个执行任务（先写测试，再实现功能）

### 阶段内执行
```bash
# 示例：执行阶段1
1. 开始子代理
2. 执行Task 1-4
3. 每个任务都要：
   - 写测试
   - 实现功能
   - 运行测试
   - 确认通过
4. 阶段完成后：
   - 检查所有功能
   - git add .
   - git commit -m "feat: complete phase 1 - data models"
```

### 阶段转换
1. 当前阶段所有任务完成
2. 运行完整测试套件
3. 提交代码到 git
4. 记录执行进度
5. 清理会话：
   - 保留关键决策
   - 删除中间输出
6. 休息 5-10 分钟
7. 准备下一阶段

## 注意事项

### 上下文管理
- 每个阶段保持专注，不要引入无关任务
- 使用简洁的代码注释，避免冗长解释
- 及时清理不再使用的变量和对象

### 代码质量
- 严格遵循 TDD 原则
- 保持 DRY（Don't Repeat Yourself）
- 遵循 YAGNI（You Ain't Gonna Need It）
- 每个任务都要有对应的测试

### 版本控制
- 每个阶段完成后必须提交
- 提交信息要清晰描述阶段内容
- 可以考虑创建分支来管理不同阶段

### 风险控制
- 遇到困难时及时暂停
- 记录问题点和解决方案
- 必要时调整任务顺序
- 优先保证核心功能

## 应急方案

### 任务卡住时
1. 检查是否理解正确
2. 查看相关文档
3. 简化问题，分步解决
4. 必要时跳过，后续再回访

### 发现重大问题
1. 记录问题详情
2. 暂停当前执行
3. 与相关人员讨论
4. 调整实现计划

### 进度延迟
- 评估延迟原因
- 调整任务分解
- 考虑并行处理部分任务
- 确保核心功能优先完成

---

通过这种混合策略，我们可以在保持开发效率的同时，有效管理上下文累积问题，确保项目顺利推进。
