# 会话管理指南

## 概述

本文档记录了在执行 Private Kitchen 应用更新项目时的会话管理相关问题和最佳实践，包括会话关闭和模型切换的策略。

---

## 1. 会话管理

### 1.1 是否可以随时关闭会话？

**答案：是的，您可以随时关闭会话。**

#### 关闭后的状态
- **数据保存**：
  - 已创建的文档和文件（如设计文档、实现计划等）会保留
  - 任务执行进度会被记录
  - 如果代码已提交到 git，更改会保留
  - 未提交的代码更改会丢失

#### 下次打开会话
- **继续执行**：
  - 可以使用 `resume` 参数继续之前的任务
  - 从上次离开的地方继续执行
  - 所有已创建的文件和文档仍然存在

#### 推荐做法
- **最佳关闭时机**：
  - 在阶段转换点关闭（每个阶段完成后）
  - 完成一个完整的任务模块后
  - 需要长时间离开时

- **保存步骤**：
  1. 完成当前任务/阶段
  2. 运行测试确保功能正常
  3. 提交代码到 git
  4. 记录下一步计划
  5. 关闭会话

## 2. 模型切换

### 2.1 是否可以在执行过程中切换大语言模型？

**答案：是的，可以在执行过程中切换大语言模型，但需要考虑相关影响。**

#### 切换的影响

##### 1. 内容连续性
- **问题**：不同模型的写作风格和代码风格可能不同
- **影响**：可能影响代码的一致性
- **应对**：需要重新建立上下文理解，保持风格统一

##### 2. 功能支持
- **现状**：所有模型都支持标准工具（Read、Edit、Write、Bash等）
- **差异**：不同模型的推理能力可能略有差异
- **建议**：复杂任务建议使用更强力的模型

##### 3. 性能考虑
- **速度**：Haiku > Sonnet > Opus
- **成本**：Haiku < Sonnet < Opus
- **推理能力**：Opus > Sonnet > Haiku

#### 切换建议

##### 1. 最佳切换时机
- **阶段边界**：每个阶段完成后
- **任务完成**：完成一个完整的功能模块后
- **遇到困难**：当前模型无法解决特定问题时

##### 2. 切换前准备
```bash
# 1. 保存当前进度
git add .
git commit -m "save progress before model switch"

# 2. 记录当前状态
# 在文档中记录：
# - 当前完成到哪个任务
# - 下一步计划
# - 遇到的问题和解决方案
```

##### 3. 切换后处理
- **重新理解上下文**：新模型需要时间理解项目
- **重新阅读文档**：设计文档、实现计划等
- **保持一致性**：尽量遵循之前的代码风格
- **测试验证**：切换后运行测试确保功能正常

#### 不同阶段推荐的模型

| 阶段 | 推荐模型 | 原因 |
|------|----------|------|
| 阶段1-2（基础架构） | Sonnet / Opus | 需要精确的数据模型设计 |
| 阶段3-5（界面开发） | Sonnet | 平衡性能和代码质量 |
| 阶段6（测试和文档） | Opus | 需要全面的测试覆盖和文档 |

## 3. 最佳实践总结

### 3.1 会话管理最佳实践

1. **分阶段执行**：
   - 每个阶段控制在 2-3 小时
   - 阶段之间有明确的休息
   - 避免长时间连续工作

2. **定期保存**：
   - 每完成一个任务就提交一次
   - 使用有意义的提交信息
   - 记录每个阶段的成果

3. **文档记录**：
   - 记录执行过程中的关键决策
   - 保存遇到的问题和解决方案
   - 更新项目进度文档

### 3.2 模型切换最佳实践

1. **保持一致性**：
   - 同一项目尽量使用同一模型
   - 如需切换，在阶段边界进行
   - 切换后注意代码风格统一

2. **性能优化**：
   - 简单任务使用快速模型
   - 复杂任务使用强力模型
   - 根据时间和成本权衡选择

3. **风险控制**：
   - 切换前保存完整进度
   - 切换后立即测试
   - 准备回滚方案

## 4. 紧急情况处理

### 4.1 会话意外关闭
1. 检查 git 提交记录
2. 查看最后保存的文件
3. 根据文档恢复执行
4. 如有丢失，从最近提交恢复

### 4.2 模型切换出现问题
1. 返回原模型
2. 重新建立上下文
3. 重新执行相关任务
4. 记录问题供后续参考

### 4.3 进度丢失
1. 检查文档和文件
2. 查看 git 历史
3. 重新开始相关任务
4. 更新进度跟踪

- [ ] 文档是否已更新？
- [ ] 下一步计划是否已记录？
- [ ] 相关文件是否已保存？

### 5.2 切换模型前
- [ ] 是否在阶段边界？
- [ ] 进度是否已保存？
- [ ] 代码风格是否重要？
- [ ] 是否需要更强的推理能力？
- [ ] 切换后是否需要额外解释？

### 5.3 开始新阶段
- [ ] 是否已阅读相关文档？
- [ ] 理解当前任务范围？
- [ ] 准备好测试环境？
- [ ] 明确成功标准？
- [ ] 有无需要协助的地方？

---

通过遵循这些指南，可以确保项目执行的连续性和效率，同时灵活应对各种情况。
